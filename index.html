Carte Conceptuelle sur le Cryptage

1. Cryptage (Encryption)

Définition : Processus de transformation des informations (texte, fichiers, messages) afin qu’elles deviennent illisibles pour les personnes non autorisées.

2. Concepts fondamentaux

  • Texte en clair (Plaintext): Message original, lisible. Ex: "Bonjour, comment vas-tu ?"
  • Texte crypté (Ciphertext): Message transformé, illisible sans clé. Ex: "Xjpfqz, krvmlq kxfw-xu ?"
  • Clé de cryptage (Encryption Key): Valeur utilisée pour transformer le texte en clair en texte crypté. Ex: "123"
  • Clé de décryptage (Decryption Key): Valeur utilisée pour reconvertir le texte crypté en texte lisible.
  • Algorithme de cryptage (Encryption Algorithm): Méthode mathématique pour transformer le texte en clair en texte crypté. Ex: AES, RSA, DES

3. Types de cryptage

  • Cryptage symétrique: Même clé pour cryptage et décryptage. Ex: AES. Avantage: rapide, Inconvénient: clé doit être partagée secrètement
  • Cryptage asymétrique: Clés différentes pour cryptage et décryptage (clé publique et privée). Ex: RSA. Avantage: clé publique peut être partagée, Inconvénient: plus lent

4. Avantages du cryptage

  • Confidentialité des données: Seuls les destinataires autorisés peuvent lire les messages. Ex: WhatsApp chiffré de bout en bout
  • Intégrité des données: Garantit que le message n’a pas été modifié. Ex: vérification de fichiers téléchargés avec un hash
  • Authentification: Vérifie l’identité de l’émetteur ou du destinataire. Ex: connexion sécurisée avec certificats SSL
  • Non-répudiation: Empêche l’expéditeur de nier avoir envoyé un message. Ex: signatures numériques dans les emails
